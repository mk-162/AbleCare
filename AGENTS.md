# TinaCMS Cloud indexing

`tina/tina-lock.json` is **committed** and is the schema Tina Cloud reads to
index this repo (the cloud does **not** run `tinacms build`). If you change
`tina/config.ts`, you MUST regenerate the lock (`npm run tina:lock`) and commit
it, or Tina Cloud branch indexing breaks ("indexing branch failed" /
"Unable to seed …") even though `tinacms audit` passes. CI enforces this
(`.github/workflows/tina-lock-check.yml`). Note `tina/__generated__/` is the
opposite: gitignored, regenerated by the host build — never commit it.

Full diagnosis + fix + prevention runbook (portable to other Tina projects):
**[`docs/tina-cloud-reindex-runbook.md`](docs/tina-cloud-reindex-runbook.md)**.
